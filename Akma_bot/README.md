# MBTI Диагностический Telegram Бот (Akma)

Современный telegram-бот для психологической диагностики типов личности по методике MBTI (Myers-Briggs Type Indicator) с использованием искусственного интеллекта.

## 🌟 Возможности

- **Интерактивная беседа с ИИ** - естественный диалог для определения типа личности
- **Классический режим тестирования** - стандартные вопросы с вариантами ответов
- **Демо-режим** - автоматическое тестирование с симуляцией пользователя
- **Управление сессиями** - отслеживание активности пользователей и восстановление при ошибках
- **Поддержка голоса** - озвучивание вопросов через Edge TTS, pyttsx3, Google TTS
- **Многоязычная поддержка** - русский язык с проверкой валидации ответов
- **Безопасная отправка сообщений** - обработка ошибок с повторными попытками
- **Логирование** - детальное логирование всех операций с привязкой к пользователю

## 📋 Требования

- **Python 3.12+** (✅ протестировано и работает стабильно)
- Telegram Bot Token
- OpenAI API Key или OpenRouter API Key
- Зависимости указаны в [requirements.txt](requirements.txt)

## 🚀 Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd Akma_bot
```

### 2. Создание виртуального окружения

```bash
# На Windows
python -m venv venv
venv\Scripts\activate

# На macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

pip install --upgrade edge-tts

### 4. Конфигурация окружения

Скопируйте файл `.env.example` в `.env` и заполните необходимые переменные:

```bash
cp .env.example .env
```

**Содержимое файла `.env`:**

```env
# Telegram
TELEGRAM_BOT_TOKEN=your_bot_token_here

# OpenAI / OpenRouter
OPENAI_API_KEY=your_openai_key_here
OPENROUTER_API_KEY_W=your_openrouter_key_here
```

### 5. Подготовка данных

- Положите файл с вопросами MBTI в формате Excel в папку `data/` (имя должно совпадать с `xlsx_path` в `data/config.yaml`)
- Убедитесь, что файл содержит колонки: `Dichotomy`, `Question`, `OptionA`, `KeyA`, `OptionB`, `KeyB`
- Файл с пользователями `data/users.xlsx` должен быть в папке `data/`
- База данных `data/mbti_bot.db` автоматически создается в папке `data/`

### 6. Запуск бота

```bash
python main.py
```

## ⚙️ Конфигурация

### data/config.yaml

Основные параметры бота:

```yaml
# Максимальное количество вопросов
max_qty: 4

# Оси MBTI для отображения процентов
percent: "EI, JP"

# Режимы
test: false              # false = беседа с ИИ, true = классический тест
demo: false              # Автоматическое тестирование с демо-данными

voice: true              # Озвучивание вопросов

# Параметры запросов
size_query: 75           # Максимум токенов в вопросе
size_answer: 25          # Максимум токенов в ответе
xlsx_path: "data/MBTI_questions.xlsx"

# API Конфигурация
api_base: "https://openrouter.ai/api/v1"  # или null для OpenAI

# Модели ИИ
model_Akma: "arcee-ai/trinity-large-preview:free"
temperature_Akma: 0.7

model_demo: "deepseek/deepseek-r1-0528:free"
temperature_demo: 0.0

model_analis: "arcee-ai/trinity-large-preview:free"
temperature_analis: 0.0

# Стоимость
price_query: 0.15    # $ за миллион входящих токенов
price_answer: 0.6    # $ за миллион исходящих токенов
```

## 📡 API и Интеграция

### OpenRouter API

Бот использует OpenRouter API для доступа к различным моделям ИИ:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key="ваш_api_key"
)
```

### Telegram Bot API

Используется библиотека `aiogram` (v3.0+):

```python
from aiogram import Bot, Dispatcher

bot = Bot(token="ваш_bot_token")
dp = Dispatcher()
```

## 🎯 Использование

### Для пользователей

1. Найдите бота в Telegram и нажмите `/start`
2. Введите ваш email для идентификации
3. Подтвердите ваши данные
4. Пройдите тестирование (выбираем беседу или классический тест)
5. Получите результаты в формате MBTI

### Специальные коды

- `999` - вход в режим профессионального тестирования (видны технические комментарии)
- `0` - добавить нового пользователя в базу данных

### Для администраторов

Режим `id=0` предоставляет доступ к дополнительной отладочной информации.

## 🔧 Основные компоненты

### SessionManager (`app/session_manager.py`)

Управляет активными сессиями пользователей:
- Отслеживание активности
- Circuit Breaker паттерн для проблемных пользователей
- Автоматическая очистка неактивных сессий
- Детальное логирование

### Safe Messages (`app/safe_messages.py`)

Безопасная отправка сообщений с:
- Автоматические повторы при ошибках
- Обработка разных типов ошибок
- Интеграция с SessionManager
- Подробное логирование

### MBTI Logic (`app/mbti_logic.py`)

Логика диагностики MBTI:
- Генерация вопросов с помощью ИИ
- Анализ ответов пользователя
- Озвучивание вопросов
- Формирование отчетов

### Bot (`app/bot.py`)

Основной telegram-бот:
- Обработка команд (`/start`, `/help`)
- Управление состояниями (FSM)
- Обработка ввода пользователя
- Работа с базой данных SQLite

## 📊 Формат результатов

Результаты тестирования представляются в виде четырехбуквенного кода MBTI:

- **E/I** - Экстраверсия / Интроверсия
- **S/N** - Чувствование / Интуиция
- **T/F** - Логика / Этика
- **J/P** - Суждение / Восприятие

Пример: `ENTJ` - Пол Вулф, предприниматель

## 🐛 Решение проблем

### "TELEGRAM_BOT_TOKEN не найден"

Убедитесь, что:
1. Создан файл `.env`
2. В нем установлена переменная `TELEGRAM_BOT_TOKEN`
3. Значение токена правильное и скопировано полностью

### "API-ключ не задан"

Проверьте:
1. Переменные `OPENAI_API_KEY` или `OPENROUTER_API_KEY_W` в `.env`
2. Доступ к интернету
3. Статус API сервиса

### Таймауты при отправке сообщений

- Увеличьте `timeout` в конфигурации бота
- Проверьте интернет соединение
- Убедитесь, что Telegram API доступен
